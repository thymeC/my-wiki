## 快速结论

- **Tableau Prep Builder** 是 Tableau 生态里的可视化数据准备工具，适合在分析前做数据清洗、字段筛选、格式标准化、合并、联接和输出。
- 工作方式是把数据准备过程做成一个 **flow**：输入数据源，逐步添加清洗、转换、合并步骤，最后输出为文件、发布数据源或写回支持的目标位置。
- Prep 的思路更接近「可视化 ETL」：每一步都保留在画布上，方便回看数据从哪里来、经过了哪些处理、最后输出到哪里。
- 对一次性或手动运行的数据准备，本地 Prep Builder 就能完成很多工作；如果要把 flow 放到服务器上按计划自动刷新，需要确认 Tableau Server / Tableau Cloud 侧的许可、角色和附加能力是否支持。

## 基本工作流

### 1. Add Data：连接数据源

从 **Add Data** 开始，选择需要的 data connector，例如：

- 文件：Excel、CSV、文本文件等。
- 数据库：关系型数据库、云数据库等。
- Tableau 数据源或已发布的数据资产。

选择数据源后，把需要处理的表或文件拖入画布。多个来源可以分别作为独立的数据块存在，后续再清洗、Join 或 Union。

### 2. Input Step：初步选择字段

数据接入后，先在输入步骤里确认：

- 哪些表、sheet 或字段要参与处理。
- 是否需要提前排除无关字段。
- 字段类型是否大体正确。
- 数据样本是否符合预期。

如果某些列后续完全不用，越早排除越好，能让后面的清洗和联接更清楚。

### 3. Clean Step：清洗和标准化

Clean Step 通常用于：

- 过滤行：保留需要的数据，排除异常值、空值或无关分类。
- 排除列：删除不需要的字段。
- 重命名字段：把字段名改成更容易理解的业务名称。
- 修改数据类型：例如日期、数值、字符串之间的转换。
- 分组和替换值：把写法不同但含义相同的值统一起来。
- 拆分字段：例如把一个复合字段拆成多个字段。

如果只是简单筛选字段，可以在已有步骤中完成；如果清洗逻辑变复杂，建议单独添加 Clean Step，让 flow 更容易阅读。

### 4. Join / Union：合并数据

当数据来自多个表或文件时，常见合并方式有两类：

- **Join**：横向拼接字段，适合通过 key 把事实表、维度表或不同业务表关联起来。
- **Union**：纵向追加行，适合同结构数据的合并，例如多个月份的 CSV、多个地区的同格式表。

常见操作方式是把一个数据块拖到另一个数据块上，然后选择 Join、Union 或其他可用操作。Join 后要重点检查：

- Join key 是否正确。
- Join 类型是否符合分析目的。
- 是否出现重复行、丢失行或异常 null。
- 同名字段是否需要重命名或排除。

### 5. Output：输出结果

最后添加 **Output**，决定清洗后的数据保存到哪里。常见输出包括：

- 本地文件。
- Tableau 可使用的数据文件。
- 发布到 Tableau Server / Tableau Cloud 的数据源。
- 写入支持的数据库或目标位置。

输出前建议检查字段命名、数据类型、行数、关键指标汇总值，避免把中间态数据当成最终数据发布。

## 实操顺序速记

1. **Add Data**：选择 data connector。
2. **Drag data blocks**：把需要的数据源拖入画布，保持来源分开。
3. **Filter / Exclude columns**：先过滤行、排除无用列。
4. **Add Clean Step**：需要额外清洗时再加，逻辑简单时可直接进入下一步。
5. **Join / Union / Aggregate**：按分析需要合并、追加或汇总数据。
6. **Validate**：检查行数、字段、空值、重复值和 Join 结果。
7. **Output**：添加输出步骤，生成最终数据结果。

## 注意点

- 不要把所有处理都堆在一个步骤里；关键清洗、Join、输出前检查可以拆成多个步骤，方便排错。
- Join 后优先检查行数变化，行数突然变多通常意味着 key 不唯一或 Join 条件不够准确。
- 对业务含义重要的字段，清洗后要抽样检查原始值和清洗值是否一致。
- 如果 flow 需要定时运行，提前确认许可、服务器能力、数据源凭证和刷新权限。
- 对长期复用的数据准备流程，建议给步骤命名，让每一步表达业务含义，而不是只保留默认名称。
